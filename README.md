# Payroll Tool（薪資管理系統）

## 專案簡介

本專案為一個薪資管理系統，涵蓋員工管理、考勤匯入、時薪管理、薪資計算與報表產出，
模擬實務中薪資結算流程，並以 MVP（Minimum Viable Product）為目標完成。

---

## 核心功能

### 1. 員工管理

* 員工新增 / 編輯
* 員工狀態切換（active / inactive）
* inactive 狀態的員工不參與後續功能流程

### 2. 考勤管理

* Excel 匯入考勤資料
* 支援多欄位名稱自動對應
* 依年月查詢考勤紀錄

### 3. 時薪管理

* Excel 匯入時薪資料
* 支援不同員工於不同案場設定不同時薪
* 提供查詢與手動維護

### 4. 薪資計算

* 依月份計算員工薪資
* 計算規則：

  * 各案場小計 = 時數 × 時薪
  * 總額 = 各案場小計加總
* 薪資拆分：

  * 本薪
  * 加班費（上限 5000）
  * 延長加班費
  * 本薪調整
* 僅針對「考勤 + 時薪資料完整」的員工進行計算
* 不完整資料會略過並回報

### 5. 報表產出

* 支援起訖月份查詢
* 匯出 Excel 報表
* 基於已計算薪資結果，不重新計算

---

## 系統流程

```text
員工 → 考勤 → 時薪 → 薪資計算 → 報表
```

---

## 重要設計說明

### 1. 資料完整性驗證

薪資計算建立在以下條件：

* 員工存在
* 有考勤資料
* 有對應時薪資料

若資料不完整：
→ 不進行計算
→ 納入 skippedItems 回報

---

### 2. 員工狀態

* 使用 `status = active / inactive`
* inactive 員工：

  * 不會出現在考勤、時薪、薪資流程中
  * 但仍保留歷史資料

---

### 3. 模組切分

系統拆為兩個主流程：

#### （1）人員 / 考勤流程

* 員工管理
* 考勤匯入

#### （2）薪資流程

* 時薪管理
* 薪資計算
* 報表產出

此設計用於避免範例資料不一致導致系統耦合錯誤。

---

### 4. 多場地設計

* 支援同一員工於不同案場設定不同時薪
* 薪資計算以「員工 + 場地」計算小計，再彙總

※ 由於範例資料僅提供單一場域，實際多場地情境以資料模型支援為主

## 專案啟動方式

```bash

npm run install:all

# 初始化資料庫
cd backend
npx ts-node ./src/db/init-db.ts
cd ..

# 啟動系統（前後端）
npm run dev
```

## Demo 流程（建議操作順序）

1. 建立或匯入員工資料
2. 匯入考勤資料
3. 匯入時薪資料
4. 執行薪資計算
5. 查詢薪資結果
6. 匯出報表

---

## 限制與說明

* 範例資料（考勤 / 時薪）之間存在不一致情況
* 系統採用「資料完整才計算」策略處理
* 多場地計算邏輯已支援，但範例資料無法完整驗證
